# 📱 혜핏(HyeFit) MVP 한 장 기획서 (One-Pager PRD)

## 1. 제품 한 줄 정의

**혜핏은 사용자의 카드 혜택 기준과 소비 내역을 결합해
“얼마를 더 쓰면 혜택이 적용되는지”를 알려주는 자동 금융비서 앱이다.**

---

## 2. MVP 목표

* 카드 혜택을 **실제로 받았다는 경험** 제공
* 계산·기록 없는 **혜택 중심 가계부**
* 1인 개발로 **6~8주 내 구현 가능**

---

## 3. 타겟 사용자

* 20~50대 직장인
* 카드 2~5장 보유
* 월 생활비 100~200만 원
* 가계부는 실패했지만 절약 니즈는 높음

---

## 4. 핵심 사용자 문제

1. 카드 혜택 구조가 복잡함
2. “얼마 더 써야 혜택이 되는지” 모름
3. 가계부가 귀찮아서 지속 불가

---

## 5. 핵심 해결 방식 (MVP)

* **카드 혜택 기준 DB**를 기반으로
* **소비 누적 → 혜택 구간 자동 계산**
* **공여기간 종료 전 알림**
* 수동 입력 최소화 가계부

---

## 6. MVP 핵심 기능 (In Scope)

### ① 인증

* 이메일 로그인 (Supabase Auth)

### ② 카드 관리

* 카드 선택/등록
* 사용자 카드 별명 설정

### ③ 카드 혜택 기준 (핵심)

* 카드별 혜택 Rule 저장

  * 전월 실적 기준
  * 카테고리
  * 혜택률
  * 최대 혜택
  * 혜택 구간(threshold)

### ④ 소비 입력 (간소화)

* 금액
* 날짜
* 카테고리
* 사용 카드

### ⑤ 혜택 계산

* 카드별 공여기간 누적 소비 계산
* “혜택까지 남은 금액” 표시
* 예상 혜택 금액 산출

### ⑥ 대시보드

* 오늘 추천 카드
* 카드별 혜택 진행률
* 이번 달 절약 금액

---

## 7. MVP 제외 범위 (Out of Scope)

* 마이데이터 실시간 연동
* 카드사 API
* 자동 결제 연동
* 선결제 / 가맹점
* AI 고도화

---

사용자는 “카드 + 소비”만 제공**

---

## 9. 기술 스택

* **Frontend:** Flutter
* **Style:** Tailwind 유사 유틸 개념 (Design Token)
* **Backend:** Supabase (Auth, Postgres, RLS)
* **알림:** 앱 내 알림 (Push v1 이후)
